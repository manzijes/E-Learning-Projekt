Kapitel 4

Media Queries schreiben

Lernziele:

Du kannst verschiedene Arten von Media Queries definieren.

Wir machen gute Fortschritte, findest du nicht?

Aber jetzt nochmal aufgepasst, es wird wichtig! Ich zeige dir nun, was Media Queries sind und wie man sie verwendet. Also Lauscher auf, ja?

Wozu Media Queries?

Mit Media Queries kannst du Ausgabegeräte auf bestimmte Ausdrücke testen. Bei true, also wenn der Ausdruck zutrifft, wird der CSS-Anwendungsblock in den geschweiften Klammern des Media Queries verwendet.

Schema

Media Queries sehen schematisch so aus.

                            @media [not|only] type and (expression) [AND (expression)] {
                                /* CSS-Anweisungen */
                            }

Komponenten

Media Queries setzen sich aus mehreren Komponenten zusammen und die beiden Hauptkomponenten sind:

  1. Das Schlüsselwort für den Medientyp. Ein Medientyp kann für sich alleine stehen oder es können mehrere Typen mit einem Komma separiert gelistet werden.
  2. Der Ausdruck für die Medieneigenschaften. Diese stehen immer in runden Klammern und es können auch mehrere Medieneigenschaften kommasepariert aufgelistet werden.

Mehrere Typen und Eigenschaften

So sieht ein Media Query mit mehreren Typen und Eigenschaften aus.

                            @media [not|only] type1, type2, and (expr1, expr2, expr3) {
                                /* CSS-Anweisungen */
                            }

Neben den beiden Hauptkomponenten gibt es auch logische Schlüsselwörter.

Schlüsselwörter

Die optionalen Schlüsselwörter (key words) not oder only kannst du einleitend vor den Medientyp stellen: Mit not wird der gesamte Ausdruck eines Media Querys negiert und somit wird das Stylesheet nicht interpretiert, wenn der Rest der Media Querys nicht zutrifft. Mit only kannst du Stylesheets mit Media Querys vor älteren Browsern verbergen. Mit Verwendung dieser Schlüsselwörter stellst du sicher, dass Ausgabemedien, die nicht die Medieneigenschaften interpretieren, das Stylesheet niemals laden.

Medientypen und Medieneigenschaften kombinieren...

                            @media [not|only] type1 and (expr1), type2 and (expr2), type3 and (expr3) {
                                /* CSS-Anweisungen */
                            }

...durch Kommata

Ein Komma zwischen Typeneigenschaften wirkt wie ein logisches or. Das bedeutet, nur einer der Ausdrücke muss zutreffen, damit der gesamte Ausdruck wahr ist.

Medientypen und Medieneigenschaften kombinieren...

                            @media [not|only] type1 and (expr1) and (expr2) and (expr3){
                                /* CSS-Anweisungen */
                            }

...durch and

Du kannst auf verschiedene Eigenschaften prüfen, indem du diese mit and verbindest. Die Styles werden nur dann verwendet, wenn type1 und jede der Ausdrücke zutreffen. Steht not davor, werden die Styles nicht ausgeführt.

CSS-Eigenschaften

Die letzte Komponente sind die CSS-Eigenschaften, die in der geschweiften Klammer stehen.

                            /* CSS-Anweisungen für Basisversion */

                            @media screen and (min-width: 720px) {
                            /* CSS-Anweisungen */
                            }
                            
                            @media screen and (min-width: 986px) {
                            /* CSS-Anweisungen */
                            }
                            
                            @media screen and (min-width: 1200px) {
                            /* CSS-Anweisungen */
                            }

Du kannst Blöcke von CSS-Deklarationen direkt im Stylesheet definieren, die nur unter bestimmten Bedingungen ausgeführt werden.

Oder du kannst auch im HTML-Code komplette Stylesheets über link-Elemente zuweisen.
